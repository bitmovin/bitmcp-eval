A recording proxy, not a rewrite

bitmcp-eval runs your prompts through a real chat agent, intercepts every call to your MCP server, and checks the observed tool calls against your expectations — multiple times per prompt, because agent behavior has spread.

📝

Your prompts

YAML test cases with the tools you expect the agent to call.

→
🤖

Real chat agent

claude or codex, run headlessly per iteration.

→
🛰️

Recording proxy

Forwards MCP traffic byte-for-byte and records every tool call.

→
🗄️

Your MCP server

Unchanged. Anything speaking MCP StreamableHTTP works.

The proxy operates on the transport layer — zero changes to your server — then validates expected vs. recorded tool calls into a live TUI and a static HTML report.

Quickstart

Two minutes against the bundled demo weather server, to verify your toolchain end to end.

# install & build the monorepo
corepack enable
yarn install
yarn build

# terminal 1 — demo MCP server on :3210
yarn demo-server

# terminal 2 — run the evaluation
yarn start -c examples/eval.yaml

What's in the box

Everything needed to turn "it seemed to work" into a measured pass rate.

🛰️

Transport-level proxy

Records every tools/call — name, arguments, result, error state, duration — with no changes to the server under test.

🤝

claude & codex

Run one agent or both. The suite executes once per agent with per-agent pass rates — "does codex use my server as reliably as claude?"

🔁

Iterations & spread

Each test case runs N times, because agent behavior varies. You see 7/10, not a single lucky green check.

🔐

OAuth, handled

401 is auto-detected; the proxy does authorization-code + PKCE, caches and refreshes tokens. Dynamic client registration needs zero config.

⚖️

Optional LLM judge

A second, semantic opinion per iteration (any OpenAI-compatible endpoint). It never overrides the mechanical result — disagreements get flagged.

📊

Live TUI + HTML report

Watch runs live, then share a self-contained report with each iteration's tool calls, arguments, missing expectations, and full conversation.
