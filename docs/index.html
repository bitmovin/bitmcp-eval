A recording proxy, not a rewrite

bitmcp-eval starts a proxy in front of your server and points the agent at it. Every JSON-RPC message is forwarded byte-for-byte while a copy records each tools/call — name, arguments, result, error, duration. It runs on the transport layer, so the server under test needs zero changes.

pipeline.txt 
  prompt          MCP / StreamableHTTP            forward + record

┌─────────────┐      ┌──────────────┐      ┌──────────────────┐      ┌─────────────┐

bitmcp-eval --features

What's in the box

transport-level proxy

Records every tools/call with args, result, error state and duration — no changes to the server under test.

claude & codex

Run one agent or both; the suite executes once per agent with per-agent pass rates. Does codex use your server as reliably as claude?

iterations & spread

Each test case runs N times because agents are non-deterministic. You measure the spread, not a single run.

oauth, handled

A 401 is auto-detected; the proxy runs auth-code + PKCE, caches and refreshes the token. Dynamic client registration needs zero config.

optional llm judge

A semantic verdict per iteration from any OpenAI-compatible endpoint. Never overrides the mechanical result — disagreements get flagged.

live tui + html report

Watch runs live, then share a self-contained report with each iteration's tool calls, missing expectations and full conversation.
